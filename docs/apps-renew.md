## Summary

- **20 apps** now have **progenitor-generated clients in ext (shared mode)**: binance, bybit, okx, dune, defillama, neynar, oneinch, hyperliquid, dydx, gmx, kalshi, zerox, cow, lifi, across, yearn, manifold, x, khalani, kaito.
- **3 apps** with **app-local generated clients**: delta, para, pelagos.
- **2 kept hand-written by design**:
  - **morpho** — GraphQL API; OpenAPI doesn't model it.
  - **polymarket** — uses official `polymarket-client-sdk` + alloy signers for L1/L2 EIP-712 signing across 3 APIs (Gamma/CLOB/Data); not modelable in OpenAPI.
- **3 HMAC-signed apps** (binance, bybit, okx) preserve hand-written `ext/src/<app>/auth.rs` shims that compute signatures and pass them to the generated client.
- **All 25 curated apps still build** with user-centric tool layers.
- **1 app with full e2e test coverage**: khalani.
- **6 apps in the long tail** (admin, byreal, l2beat, molinar, para-consumer, polymarket-rewards) are still uncurated.
- **Removed**: `apps/{defi, prediction, social, test-dyn}/` deleted — also pruned from workspace exclude list and `aomi/plugins/manifest.json`.

## Bonus findings during this round

- **dydx**: spec drafting verified two latent bugs in the old hand-written client (`/historical-funding/` should be `/historicalFunding/`; `/candles/perpetualMarket/` should be `/candles/perpetualMarkets/`). Fixed via the new spec.
- **cow**: official upstream spec at `https://raw.githubusercontent.com/cowprotocol/services/main/crates/orderbook/openapi.yml` — much higher fidelity than skill-drafted alternatives.
- **lifi**: previous spec was wrongly discovered (apis-guru returned `intellifi.nl`, an unrelated IoT API). Replaced with skill draft.
- **x**: previous spec was wrongly discovered (Box.com). Replaced with skill draft for twitterapi.io.
- **hyperliquid**: was flagged as a bad-fit case; agent successfully modeled the single `POST /info` endpoint with a free-form JSON body, getting a ~150-line client comparable to hand-written.
