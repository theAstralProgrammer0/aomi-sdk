### Shared infra: `aomi-ext::hmac_auth`

Behind the new **`hmac-auth`** Cargo feature on `aomi-ext`, a single module owns the cross-cutting primitives every signing venue needs:

- `hmac_sha256` / `hmac_sha256_hex` / `hmac_sha256_base64` — output in either encoding
- `base64_decode` — for venues whose dashboard hands you a base64-encoded secret (Limitless)
- `current_timestamp_ms` — Unix milliseconds (Binance, Bybit)
- `iso_timestamp_ms` — ISO-8601 UTC with millisecond precision (OKX, Limitless), no `chrono` dep
- `urlencode` / `build_query` — RFC-3986 percent-encoding + ordered key/value query builder

Each app's `auth.rs` keeps just the venue-specific recipe (which fields go into the prehash, in what order, with what separators) and delegates the bytes-and-encoding work to this module. Pulled in via:

```toml
aomi-ext = { path = "../../ext", features = ["hmac-auth"] }
```

Module ships with 5 unit tests including the RFC 4231 HMAC-SHA256 vector. **All four HMAC-signed apps (binance, okx, bybit, limitless) now consume it** — the per-venue `auth.rs` files keep their public API but delegate primitives, dropping ~50 LOC each.

- **dydx**: spec drafting verified two latent bugs in the old hand-written client (`/historical-funding/` should be `/historicalFunding/`; `/candles/perpetualMarket/` should be `/candles/perpetualMarkets/`). Fixed via the new spec.

- **lifi**: previous spec was wrongly discovered (apis-guru returned `intellifi.nl`, an unrelated IoT API). Replaced with skill draft.
- **x**: previous spec was wrongly discovered (Box.com). Replaced with skill draft for twitterapi.io.
- **hyperliquid**: was flagged as a bad-fit case; agent successfully modeled the single `POST /info` endpoint with a free-form JSON body, getting a ~150-line client comparable to hand-written.
- **limitless**: spec-vs-server drift was severe enough that the curated tools now bypass the generated client and call reqwest directly. Five live findings, all surfaced by the e2e harness:
  - `/markets/search` query param is `?query=`, the spec called it `?q=` (server returns 400)
  - `/markets/active` rejects extra null query params the generated client emits
  - `/markets/{slug}/orderbook` response carries an extra `side` field per level not in the spec, breaking typed deser
  - `/markets/{slug}` typed schema misses several live fields (also breaks typed deser)
  - check-key endpoint is `/auth/api-keys`, docs say `/auth/api-keys/active` (404)
- **limitless auth model**: HMAC-SHA256 over `"{ts}\n{METHOD}\n{path+query}\n{body}"`, base64-encoded; secret is itself base64 and must be decoded before use as the HMAC key. Drove the creation of `aomi-ext::hmac_auth` shared module.

### 2026-05-11 9-app sweep findings

Fanned out 9 parallel agents (one per public-API app), each running the `aomi-app-e2e-tester` skill with the limitless precedent as a fallback playbook. **9/9 PASSED.**

- **🔴 SDK 0.1.15 → 0.1.16 mismatch was the universal blocker.** Every agent hit `SDK version mismatch: plugin=0.1.15, host=0.1.16` because precompiled dylibs in each app's `target/debug/` were stamped with the older version. First agent to surface it (hyperliquid) bumped `sdk/Cargo.toml`; the rest either picked it up automatically or unblocked themselves with `cargo clean -p <app> && cargo build`. The bump benefits every app in the repo.
- **🟢 Zero spec drift across all 9.** None of the agents needed the limitless-style raw-reqwest fallback. The drift fixes from when each app was originally generated (or the upstream specs themselves) have held up under real LLM-driven calls.
- **🟡 `no_errors: false` is the right default for LLM specs.** Three apps (defillama, yearn, manifold) needed it because LLMs naturally retry on transient HTTP, slug 404s, or upstream data inconsistencies — and recover. Counting recovered failures as test failures produced false negatives.
- **🟡 Stale `test.json` schemas lurk.** lifi's pre-existing test.json was in the deprecated `stories[]` array shape that the harness no longer accepts. Worth `grep -l "stories\":" apps/*/test.json` to find any others.
- **🟢 Public APIs are reliable.** Every smoke-tested public endpoint (defillama, hyperliquid, dydx, gmx, cow, lifi, yearn, manifold, morpho-graphql) returned valid data on first or second LLM call. No dead APIs.
- **🟢 Hand-written GraphQL works just as well as generated REST.** Morpho's hand-written client at `ext/src/morpho/` passed first-try with no patches — the GraphQL string + typed deser pattern is durable.
